## V1 — In-Band MCTF: Temporal Prediction in Wavelet Domain

**Source**: "Learned Wavelet Video Coding using MCTF" (arXiv 2305.16211, 2023) — 21% BD-rate savings over HEVC by applying temporal lifting *after* spatial DWT, not before.

**Proposal**: Flip the pipeline so that spatial wavelet is applied first, then temporal prediction (motion estimation and compensation) happens on subband coefficients directly — not on raw pixels. Each subband (LL, LH, HL, HH) gets its own motion field at its own resolution. P/B reference frames stored as already-transformed subbands.

**Why it might help GNC**: Currently GNC predicts in pixel space, then transforms the residual. P-frames on crowd_run approach ~98% of I-frame size — very weak temporal decorrelation. MCTF can exploit that high-frequency subbands (HH) in static regions are near-zero and need no temporal prediction at all — spend bits only on subbands with actual temporal energy.

**Lateral leap**: HIGH — inverts the pipeline's temporal/transform stage ordering entirely.

**References**:
- https://arxiv.org/abs/2305.16211
- https://arxiv.org/abs/2405.12631

## V2 — Opus-Style Per-Tile Mode Switching

**Source**: The Opus audio codec dynamically switches between SILK (time-domain, good for speech) and CELT/MDCT (frequency-domain, good for music) on a per-frame basis.

**Proposal**: Per-tile, per-subband mode switch in GNC. For tiles with high spatial entropy (textures, edges, HH-dominant): current CDF 9/7 + Rice. For tiles that are LL-dominant (smooth gradients, camera-motion backgrounds): switch to a very-low-level wavelet or DCT just for that tile. Crossover criterion: ratio of LL energy to total subband energy.

**Why it might help GNC**: Experiment #41 found LL-dominant residuals in P-frames — the wrong tool is being used for the wrong content. Opus solves exactly this in audio. The cost of the mode bit is paid back quickly on smooth tiles where the wavelet's 4-level overhead wastes bits.

**Lateral leap**: MEDIUM — borrows a proven audio architecture decision, reframes it as a spatial codec mode switch.

**References**:
- https://arxiv.org/abs/1602.04845

## V3 — Sigma-Delta Noise Shaping in Wavelet Quantizer

**Source**: Sigma-delta ADC design + 2023 Oxford paper (*Information and Inference*) formalizing noise shaping for finite-frame transform coefficients.

**Proposal**: After quantizing each wavelet coefficient, compute the quantization error and add a fraction of it to the *spatially adjacent* coefficient before that coefficient is quantized. Error propagates in raster order within each subband. Result: quantization noise is steered toward high-spatial-frequency coefficients (HH), which are perceptually less visible, away from LL and low-frequency subbands where errors are most visible.

**Why it might help GNC**: GNC's perceptual subband weights already do something related at the step-size level, but quantization error is discarded. Noise shaping preserves total energy while reshaping *where* distortion lands — a potentially free perceptual quality improvement with no rate cost and no bitstream format change. VMAF is sensitive to exactly the low-frequency errors this would reduce.

**Lateral leap**: HIGH — an audio/ADC technique (sigma-delta) transplanted into the wavelet quantizer loop, orthogonal to anything currently in GNC.

**References**:
- https://academic.oup.com/imaiai/article/13/1/iaad052/7504772

## V4 — Channel-Wise Subband k Prediction

**Source**: Minnen et al. channel-wise autoregressive entropy model (ICIP 2020) and learned wavelet coding (arXiv 2405.12631, 2024): divide latent tensor into channel slices, use each decoded slice to condition the Rice parameter k for the next slice. Only 4 coding steps needed — 350× decode speedup vs sequential autoregressive.

**Proposal**: Code GNC's 8 subband groups in a fixed causal order (LL first, finest detail last). Use the already-coded LL Rice-k value to predict the initial k for LH/HL. Then use LH+HL to predict k for HH, level by level. This is a purely statistical prior — no neural net. The k prediction table is a 16-entry lookup built from offline statistics.

**Why it might help GNC**: MEMORY notes that parent-child entropy coding (LL predicts LH/HL/HH probability context) is exactly the mechanism VC-2 uses to approach H.264 compression — estimated gain +0.10–0.18 bpp. This idea takes the *parallelism structure* from learned codecs to implement that gain while keeping the number of GPU coding passes fixed.

**Lateral leap**: MEDIUM — steals parallelism architecture from learned compression, applies it to a classical Golomb-Rice coder.

**References**:
- https://ieeexplore.ieee.org/document/09190935/
- https://arxiv.org/abs/2405.12631

## V5 — Odd Tile Size for Boundary Artifact Reduction

**Source**: JPEG 2000 tile boundary research: tiles of size (2N+1) instead of 2N produce fewer boundary artifacts because symmetric extension interacts more cleanly with odd-dimensional DWT lifts.

**Proposal**: Change GNC's tile size from 256×256 to 255×255 or 257×257. Or: apply one-pixel asymmetric boundary extension (half-sample symmetric rather than whole-sample) at tile edges. Tiny change at tile setup stage only.

**Why it might help GNC**: Tile-edge artifacts were quantified as a 0.66 dB PSNR gap (#47). The full overlapping tile approach (Approach A) is deferred due to larger coefficient buffer requirement. The odd-size insight may recover a fraction of that 0.66 dB for nearly zero implementation cost — worth checking before building Approach A.

**Lateral leap**: LOW — incremental, but non-obvious.

**References**:
- https://airccj.org/CSCP/vol2/csit2126.pdf

## V6 — 3D Wavelet Across Color Planes (Hyperspectral Trick)

**Source**: Hyperspectral satellite imagery compression adapts SPIHT/EZW to anisotropic 3D wavelets, treating the spectral (wavelength) axis as a third spatial dimension with different decomposition depth than spatial axes.

**Proposal**: Treat GNC's Y, Co, Cr planes not as three independent 2D wavelet transforms but as a 3-"spectral"-sample input to a 1D wavelet along the color axis, applied *after* the 2D spatial wavelet. One level of Haar across the (Y, Co+Cr) triplet produces a "color-average" and "color-difference" plane jointly decorrelated. Color-difference plane's wavelet coefficients coded with shared k derived from color-average.

**Why it might help GNC**: CfL (Chroma from Luma) already does something like this at quantization time. A full 3D wavelet applies the cross-color decorrelation *before* entropy coding, potentially yielding sparser coefficient histograms in all three planes simultaneously.

**Lateral leap**: HIGH — borrows from satellite hyperspectral imaging to propose a cross-plane wavelet GNC's pipeline doesn't currently consider.

**References**:
- https://www.researchgate.net/publication/23469148

## V7 — Geometry-Oriented ZRL Scan

**Source**: Distributed acoustic sensing (DAS) compression (JGR: Machine Learning 2024) exploits structural sparsity of wavelet coefficients — energy concentrates in predictable wavefront patterns describable by compact parametric models.

**Proposal**: In GNC's significance map, replace raster-order ZRL with a *geometry-oriented* scan. Significant-coefficient clusters in HH subbands are typically edge-shaped (1D structure). Estimate dominant gradient direction cheaply from the LL subband, then orient the ZRL scan along that direction for each tile's HH subband. Different scan orders for different subbands.

**Why it might help GNC**: ZRL already addresses 1D runs, but significance patterns in wavelet subbands are 2D and geometrically structured. Orienting the scan direction per-subband based on dominant gradient could reduce significance map coding cost — similar to what SPECK does with block-split structure, but via scan-order adaptation only.

**Lateral leap**: MEDIUM — transfers geophysical wavefield sparsity insight into scan-order optimization.

**References**:
- https://agupubs.onlinelibrary.wiley.com/doi/full/10.1029/2024JH000247

## V8 — ECTCQ-Style Trellis AQ (Entropy-Driven Quantization)

**Source**: Entropy-constrained trellis-coded quantization (Marcellin & Fischer, IEEE Trans. 1990). Jointly optimizes quantization *and* entropy simultaneously via Viterbi over a trellis. Renewed interest via neural codec "entropy-constrained rate-distortion" end-to-end training.

**Proposal**: Invert the current AQ flow. Instead of computing subband step sizes first then entropy coding, run a fast one-pass trellis over each Rice group (16 coefficients) choosing between two candidate step sizes (coarser or finer than nominal AQ step). Trellis state tracks estimated bit cost; Viterbi selects the fine/coarse sequence that hits target bit budget per tile. Computable in a GPU shader: 16 steps, 2-state trellis, embarrassingly parallel per tile.

**Why it might help GNC**: GNC's AQ is a fixed per-subband weight table — doesn't react to actual entropy of coefficients in a given tile at encode time. ECTCQ-style adaptation closes that loop. Learned codecs take 10–15% BD-rate from this; classical trellis gets most of that gain without a neural net.

**Lateral leap**: HIGH — reorders pipeline stages (entropy knowledge drives quantization choices, not the other way around).

**References**:
- https://ieeexplore.ieee.org/document/119697
